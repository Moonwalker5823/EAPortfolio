Desktop Section Layout — Which approach?

The goal: click a nav link → only that section is visible. Three ways to get there.

A

100vh + CSS Scroll Snap

Each section is exactly height: 100vh. The page uses scroll-snap-type: y mandatory so scrolling always lands on a clean section boundary — no half-sections visible. Nav clicks scroll exactly to the section.

Pros

  • Silky, intentional scroll feel
  • Nav links always show exactly one section
  • Very predictable layout

Cons

  • Scroll feels "locked" — some users dislike it
  • Sections with lots of content (Contact, Portfolio) may need internal scroll
B

100vh per section, free scroll

Each section gets min-height: 100vh with content centered. Normal free scrolling — no snapping. Nav clicks jump to the section anchor. Each section fills exactly one screen on desktop.

Pros

  • Natural scrolling feel
  • Nav clicks land at the right place
  • Content-heavy sections can grow if needed

Cons

  • Fast scrollers may see two sections at once between anchors
  • Requires careful vertical centering per section
C

Fixed gap between sections (no 100vh)

Keep sections content-sized but enforce a consistent margin-top gap (e.g. 6rem) between every section. Doesn't guarantee "one section per screen" but keeps spacing uniform and predictable.

Pros

  • Simplest change — just normalize margins
  • Works naturally on all screen sizes
  • No risk of awkward empty space

Cons

  • Nav links may land mid-section on some screen heights
  • Doesn't fulfill "one section per screen" goal

My recommendation: Option B — 100vh per section, free scroll. Gives you the full-screen section behavior without locking the scroll wheel. Works great with the existing scroll-margin-top already set on some sections.
